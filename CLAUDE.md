**shadcn primitive library ‚Äî read before building any interactive component**: this project's `shadcn init` picked the `base-nova` style, built on **Base UI** (`@base-ui/react`), not Radix. This changes real APIs you'll otherwise get wrong from Radix-based shadcn knowledge/muscle memory:

- No `asChild` prop. Use `render={<YourElement />}` (or a render-function) instead ‚Äî e.g. `<DialogTrigger render={<Button>Abrir</Button>} />`, not `<DialogTrigger asChild><Button>...</Button></DialogTrigger>`. Passing `asChild` leaks it onto the DOM node as an invalid attribute (React will warn, not error ‚Äî easy to miss). For internal Next.js routes, prefer `render={<Link href="..." />}` over a plain `href` prop so navigation stays client-side (`NavigationMenuLink`/`Button`/`SheetClose` all accept `href` directly too since they render `<a>`/native elements, but that forces a full page reload).
- Accordion has no `type="single"|"multiple"` + `collapsible` props. It takes `multiple?: boolean` (default `false`) and `value`/`defaultValue` as an **array**, e.g. `<Accordion defaultValue={["item-1"]}>`.
- `NavigationMenu` (used for the mega menu) needs no `viewport` prop ‚Äî unlike Radix, the Base UI version always renders its shared positioner/viewport internally.
- When in doubt, check the type at `node_modules/@base-ui/react/<component>/**/*.d.ts` rather than assuming Radix conventions.

**`lucide-react` in this project is a `^1.x` release that dropped all brand/logo icons** (GitHub, Instagram, LinkedIn, etc. ‚Äî Lucide's own trademark-driven decision), even though generic icons are all still present. Cap√≠tulo 17 forbids mixing icon libraries, so don't reach for `react-icons`/`simple-icons` ‚Äî hand-write small inline SVGs instead, as done in `src/components/icons/social.tsx`. Some (not all) generic icons are also exported twice, once bare and once `*Icon`-suffixed (e.g. `Menu`/`MenuIcon`) ‚Äî either works, shadcn's own generated files use the suffixed form.

**Components pulled from third-party registries (React Bits, Magic UI) are vendored into this repo as regular source** (`src/components/backgrounds/DotGrid.{tsx,css}`, `src/components/animations/SplitText.tsx`, `src/components/ui/animated-beam.tsx`) ‚Äî they don't ship lint-clean for this project's config and were hand-fixed after install (untyped `any`, unused-expression, and `react-hooks/set-state-in-effect` violations). If you pull in another React Bits/Magic UI component via `npx shadcn add <registry-url>`, expect to run `npm run lint` immediately after and fix the same categories of issue rather than assuming vendored code is exempt ‚Äî Cap√≠tulo 28's Definition of Done ("sem erros de lint") applies to it too.

**`react-hooks/refs` false positive**: passing `ref={someRefsObject.someKey}` (dot-notation property access on a plain object of `useRef()` values, built inline in the component and read directly in JSX) trips the `react-hooks/refs` ESLint rule ("Cannot access ref value during render"), even though this is just selecting which ref to forward, not reading `.current`. Bracket-notation access on the same kind of object _inside a `.map()`_ (e.g. `nodeRefs[node.id]`, as used in `Ecosystem.tsx`/`InteractiveDemo.tsx`'s beam connections) does not trigger it. When placing a handful of ref'd elements individually (not mapped), just use the individual `useRef()` variables directly instead of routing through a lookup object ‚Äî see `InteractiveDemo.tsx`.

## Design system (Cap√≠tulo 17)

- Style: "Dark Premium" ‚Äî high whitespace, minimal text, interface-forward, never decorative-for-its-own-sake.
- Grid: desktop reference width `1440px`, container `max-width: 1280px`. Padding: desktop `120px`, tablet `64px`, mobile `24px`.
- Spacing scale (8px-based, never arbitrary values): `4 8 16 24 32 40 48 64 80 96 120 160`.
- Radius scale: small `8px`, medium `16px`, large `24px`, hero `32px` ‚Äî implemented as Tailwind utilities `rounded-sm/md/lg/xl` via `--radius-*` in `src/app/globals.css` (superseding Cap√≠tulo 1's single `18px` value, per resolved decisions above).
- Borders: always `1px solid rgba(255,255,255,.08)`. Shadows always very light ‚Äî never heavy.
- Colors: background `#09090B`, surface `#111113`, surface hover `#17171A`, primary `#6C63FF`, primary hover `#7A73FF`, secondary `#8B5CF6`, success `#22C55E`, warning `#F59E0B`, danger `#EF4444`. Text: primary `#FFFFFF`, secondary `#A1A1AA`, muted `#71717A`.
- Typography: Geist primary / Inter fallback (`next/font/google`, wired in `src/app/layout.tsx`). Scale ‚Äî hero `72px`, t√≠tulo `48px`, subt√≠tulo `24px`, texto `18px`, descri√ß√£o `16px`, legenda `14px` ‚Äî implemented as Tailwind utilities `text-hero/title/subtitle/body/description/caption` via `--text-*` in `src/app/globals.css`.
- All colors above are implemented as CSS custom properties in `src/app/globals.css` (`:root`/`.dark`, kept identical since the PRD never specifies a light mode) and consumed through standard shadcn/Tailwind tokens (`bg-background`, `text-muted-foreground`, `bg-primary`, etc.) ‚Äî never hardcode hex values in components.
- Buttons: height `48px`, radius `14px`, horizontal padding `24px`. Primary = filled with primary color + white text + subtle hover grow/shadow. Secondary = transparent + border, surface-hover on hover. Implemented in `src/components/ui/button.tsx` ‚Äî `size="default"` carries the spec; `variant="default"` is primary, `variant="outline"` is the PRD's "Bot√£o Secund√°rio" (not shadcn's `variant="secondary"`, which is a separate filled-accent style using the `#8B5CF6` token, not one of the two PRD button types). Has an `isLoading` prop for the loading state.
- Cards: consistent structure (icon ‚Üí title ‚Üí description ‚Üí action) ‚Äî never a bespoke layout without justification.
- Icons: Lucide exclusively, sizes `20 / 24 / 32` ‚Äî never mix icon libraries.
- Motion: entrance = fade + translateY 16px; hover = scale 1.02; click = scale 0.98; durations `200‚Äì400ms`, **never above 600ms**; easing always smooth, never "elastic"; scroll-triggered animations fire once, never loop.
- Component-library responsibility split: base components ‚Üí shadcn/ui; animation/transition logic ‚Üí Motion; ambient backgrounds ‚Üí React Bits; connective/flow diagrams ‚Üí Magic UI; hover micro-interactions ‚Üí Aceternity UI.

- Each page section answers exactly one visitor question, never two (Hero ‚Üí "who are you?", Ecosystem ‚Üí "how does it work?", Portfolio ‚Üí "what have you built?", CTA ‚Üí "how do I hire you?").
