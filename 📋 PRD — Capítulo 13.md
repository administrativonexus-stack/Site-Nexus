📋 PRD — Capítulo 13
Portfólio
Objetivo

O Portfólio deve demonstrar a capacidade técnica da Nexus através de projetos reais.

Não será apenas uma galeria de imagens.

Cada projeto deverá contar uma pequena história:

qual era o problema;
qual solução foi desenvolvida;
quais tecnologias foram utilizadas;
qual foi o resultado.
Objetivo Psicológico

Ao terminar esta seção, o visitante deve pensar:

"Eles desenvolvem produtos completos."

e não

"Eles fazem sites."

Critérios de Aceitação

A seção será considerada pronta quando:

O visitante identificar rapidamente os tipos de projetos desenvolvidos.
Os cards estiverem visualmente consistentes.
As animações forem suaves.
O carregamento permanecer rápido.
Cada projeto possuir uma página dedicada ou estrutura preparada para isso.
Estratégia de Conteúdo

Mesmo que um projeto ainda não esteja público, o componente deve ser preparado para suportar:

imagens;
vídeos;
depoimentos;
métricas (quando autorizadas pelo cliente);
tecnologias;
status do projeto.

Isso evita mudanças estruturais futuras.
